## 🚀 Descripción

Implementación de múltiples **estructuras de datos fundamentales** utilizando **JavaScript**, enfocada en comprender su comportamiento interno, lógica de almacenamiento y mecanismos de acceso.

El proyecto profundiza en cómo estas estructuras gestionan memoria, referencias y operaciones, aproximando conceptos de bajo nivel como punteros y direccionamiento dentro del contexto del motor de JavaScript.

## 🎯 Objetivos

- Implementar estructuras de datos desde cero  
- Comprender la lógica interna de almacenamiento y acceso  
- Simular el uso de punteros mediante referencias en JavaScript  
- Analizar el comportamiento del garbage collector  
- Fortalecer el pensamiento algorítmico  

## 🛠️ Tecnologías Utilizadas

- JavaScript (ES6+)  

## 📚 Estructuras Implementadas

- **ArrayList (listas dinámicas)**  
- **Hash Tables (tablas hash)**  
- **Linked List (simples y dobles)**  
- **Stacks (pilas)**  
- **Queues (colas)**  
- **Trees (árboles)**  
- **Graphs (grafos)**  

## ⚙️ Conceptos Clave

- Manejo de referencias (simulación de punteros)  
- Direccionamiento lógico en memoria  
- Complejidad temporal y espacial (Big-O)  
- Inserción, eliminación y búsqueda eficiente  
- Gestión de memoria en entornos con garbage collector  

## 🧩 Enfoque Técnico

Aunque JavaScript no expone punteros de forma explícita, este proyecto modela estructuras utilizando **referencias a objetos**, lo que permite aproximar el comportamiento de estructuras tradicionales de bajo nivel.

Se analiza cómo:

- Los nodos se enlazan mediante referencias  
- Se gestionan ciclos de vida de objetos  
- El **garbage collector** libera memoria cuando no existen referencias activas
